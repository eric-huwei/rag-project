# 拼音同音匹配规则

## 目标

本规则用于统一地址抽取流程中“重合度”“拼接”“纠错”“确认话术”“记录地址信息”的判定标准，避免测试场景与实现理解不一致。

## 总原则

1. 地址是否“有重合度”，不再只看字面是否相同，也要看拼音是否可视为同一内容。
2. 同音字、前后鼻音、平翘舌导致的差异，可以参与匹配、拼接、纠错。
3. 拼音纠错只能在同一地址主体内生效，不能跨地域、跨候选乱匹配。
4. 数字、楼栋号、单元号、门牌号仍然优先按精确信息判断，不能因为拼音相似覆盖数字冲突。

## 重合度定义

### 1. 字面重合

满足以下任一条件，认为有字面重合：

- 完全相同
- 一方包含另一方
- 分词后的地址要素存在包含关系

示例：

- `中山路` 与 `中山路3号楼2101`
- `世安新城` 与 `世安新城方景小区`

### 2. 拼音重合

满足以下任一条件，认为有拼音重合：

- 同音不同字
- 平翘舌可归一后相同
- 前后鼻音可归一后相同
- 地址片段逐字拼音等价后，可在候选地址中找到连续对应片段

示例：

- `云效县` 可视为 `云霄县`
- `长山农场树东村` 可视为 `常山农场树洞村`
- `欢景小区` 可视为 `方景小区`
- `3好楼` 可视为 `3号楼`

## 当前支持的拼音归一能力

### 1. 平翘舌归一

- `zh` 与 `z`
- `ch` 与 `c`
- `sh` 与 `s`

### 2. 前后鼻音归一

- `ang` 与 `an`
- `eng` 与 `en`
- `ing` 与 `in`

### 3. 同音字归一

系统可通过两种方式识别：

- 如果环境安装了 `pypinyin`，优先使用拼音库识别
- 如果未安装 `pypinyin`，使用内置常见地址同音字表兜底

当前重点覆盖的典型字包括但不限于：

- `号/好`
- `长/常`
- `洞/东`
- `景/井`
- `莆/普`
- `霄/效`
- `方/欢`
- `著/筑/住/柱`

## 哪些逻辑使用“拼音重合”

### 1. 是否与候选地址有重合度

用于判断：

- 当前输入是否属于地址补充
- 当前输入是否应该和上一轮未匹配地址拼接
- 当前输入是否属于“完全无关”

### 2. 是否继续沿用上一轮未匹配地址

如果本轮输入与上一轮未匹配地址或候选地址存在字面/拼音重合，则允许拼接。

示例：

- 上一轮：`中山路:`
- 本轮：`3好楼`
- 结果：可拼接为 `中山路:3号楼`

### 3. 模糊纠错

在同一地址主体内，用户输入可以被纠正为候选地址中的正确字形。

示例：

- `云效县` -> `云霄县`
- `欢景小区` -> `方景小区`
- `参海民筑3号楼` -> `沧海名著3号楼`

### 4. 连续无重合判失败

如果连续两轮输入都和候选地址不存在字面或拼音重合，则可判为地址不正确。

## 地址结构词纠正规则

### 1. 结构词范围

以下地址结构词允许按拼音纠正：

- `号楼`
- `单元`
- `号院`
- `栋`
- `幢`
- `座`
- `楼`
- `室`
- `号`
- `弄`
- `里`

### 2. 结构词纠正场景

当文本前面已经形成楼栋/门牌前缀时，后面的结构词若与标准结构词同音，可直接按结构词处理。

示例：

- `3好楼` 视为 `3号楼`
- `9坐` 可视为 `9座`

### 3. 结构词纠正目的

结构词纠正主要用于：

- 让地址片段能被正确识别为楼栋/单元/门牌信息
- 让该片段进入拼接、追问、唯一匹配等正常流程

## 回复文本中的纠正规则

### 1. “我记录的地址信息是：”中的地址

如果某段内容已通过拼音匹配确定与候选地址中的某段是同一地址要素，则回复中应使用候选地址中的正确字形，而不是保留用户的错字。

示例：

- 用户：`3好楼`
- 候选：`3号楼2101`
- 回复：`我记录的地址信息是：中山路:3号楼，请您再提供下单元号及门牌号信息。`

### 2. 场景四中的纠正要求

场景四统一按以下规则理解：

- 同拼音且确认是同一地址主体时，回复中用正确地址里的字
- 不能凭空补充用户完全没有涉及的额外地址层级

示例：

- 用户：`参海民筑3号楼`
- 候选：`沧海名著3号楼352`
- 确认话术：`请问您说的是沧海名著3号楼吗？`

### 3. 不能随意扩写

即使发生纠字，也只能纠正到用户已涉及的地址范围，不能自动把未说出的上级地址全部带出来。

示例：

- 用户：`欢景小区`
- 上一轮已确定：`世安新城`
- 可输出：`世安新城方景小区`
- 不应直接扩写到完整候选地址中的所有上级行政区

## 数字与精确信息规则

### 1. 数字优先精确匹配

以下信息不按模糊拼音放宽：

- 房间号
- 单元号
- 楼栋编号
- 门牌数字

示例：

- `2101` 与 `2102` 不能因为其他部分同音而视为相同
- `3号楼` 与 `8号楼` 不能因为地名重合而视为相同

### 2. 中文数字与阿拉伯数字可归一

示例：

- `三号楼` 与 `3号楼`
- `八号楼` 与 `8号楼`

## 不能匹配的情况

以下情况即使存在局部同音，也不能视为有效匹配：

1. 跨省、市、区县存在明显冲突
2. 楼栋号、房号等精确信息冲突
3. 只有极短片段相似，但无法确认是同一地址主体
4. 只凭拼音相同，把地址跳到另一个完全不同地域

## 典型案例

### 1. 可拼接

- 上一轮：`中山路:`
- 本轮：`3好楼`
- 结果：拼接，记录为 `中山路:3号楼`

### 2. 可确认

- 用户：`参海民筑3号楼`
- 候选：`沧海名著3号楼352`
- 结果：确认为 `沧海名著3号楼`

### 3. 可纠字但不扩层级

- 上一轮：`世安新城`
- 本轮：`欢景小区`
- 结果：`请问您说的是世安新城方景小区吗？`

### 4. 不可匹配

- 候选：`中山路3号楼2101`
- 用户：`北京市`
- 结果：无重合，不拼接

## 测试维护要求

新增测试场景时，应明确写出以下三点：

1. 该用例的重合度是靠字面命中，还是靠拼音等价命中
2. 回复中是否应纠成候选地址里的正确字
3. 是否允许带出上一轮已确认的地址片段
